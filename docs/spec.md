## 検証仕様

検証処理では、以下を確認します。

* 必須項目が存在すること
* IDが空でないこと
* IDが重複していないこと
* 不明な種別が存在しないこと
* 参照先が存在すること
* 接続や依存関係に矛盾がないこと
* 対象外の値が混入していないこと
* 必須プロパティが存在すること

---

## エラー / 警告 方針

### Error

Error は、処理を続行できない問題として扱います。

例:

* 必須項目がない
* 参照先が存在しない
* データ構造が壊れている
* 実行に必要な値が欠落している

### Warning

Warning は、処理は可能だが確認が必要な問題として扱います。

例:

* 未使用のデータがある
* 到達不能な要素がある
* 推奨されない値が使われている
* 将来的に仕様変更の対象になりそうな値がある

## 動作確認項目

v0.1 では、最低限以下を確認します。

* `ビルドコマンド` が成功する
* サンプルデータを読み込める
* サンプルデータを保存できる
* 正常データでエラーが出ない
* 異常データで想定どおりエラーが出る
* 境界ケースで処理が落ちない
